# 시황 가설 (Scenario Hypothesis) — 설계 메모

> 상태: **초안 · 검토용** (구현 전)  
> 대응 백로그: [BACKLOG.md](../BACKLOG.md) `#6` 하반기 전망  
> 기존 `#9` `signals/scenario.py`(포트폴리오 MC)와 **별개** — 이름만 비슷한 다른 기능.

## 1. 한 줄 정의

시황·정책·사이클에 대한 **가설 가지(브랜치)** 를 다이어그램으로 보고,  
각 가지의 **지지도(상대 가중 %)** 와 **근거 뉴스 링크**를 붙인 뒤,  
관심 섹터 → 기존 시그널로 딥링크하는 **맥락 전용** 기능.

**시그널 엔진·페이퍼 봇 점수에는 미반영.** 독립 레이어.

## 2. 합의된 원칙 (2026-07-18)

| 항목 | 합의 |
|---|---|
| 노드 % | **지지도** — 같은 부모 아래 형제 상대가중, 합 100%. 시장 예측 확률 아님 |
| 갱신 | **하루 1회**(기존 KB 일일 수집 후) + **수동 새로고침** |
| 엔진 영향 | 없음. BUY/SELL·봇·매수 문턱·가중치 미변경 |
| 표현 | 가설·학습용. 투자 권유·수익 보장·적중률 헤드라인 금지 |
| UI | 브랜치/다이어그램. 노드 %, 호버 시 근거 뉴스 URL |

면책 상시: `가설·학습용 · 예측·투자권유 아님 · 시그널과 별개 레이어`

## 3. 사용자 플로우

1. 인사이트 탭 → 서브 `시황 가설` 진입  
2. 루트 아래 2~4개 시나리오 가지(템플릿)와 지지도 %를 다이어그램으로 확인  
3. 노드 호버/클릭 → 가정·확인 지표·근거 뉴스(제목·출처·링크)  
4. “관심 섹터”에서 밸류체인/시그널 리스트로 이동 (점수 재계산 없음)

## 4. UI

### 4.1 배치

- **새 탭 금지** — 인사이트(`view-cycle`) 서브탭 추가 (APT 나침반: 탭 증식 금지).  
- 기존 `#6` 메모의 “시그널 오버레이”보다 **인사이트**가 맞음(맥락 레이어·사이클/밸류체인 인접).

### 4.2 다이어그램

- 형태: **트리/브랜치** (루트 → 시나리오 → 관심 섹터).  
- 구현 후보: ECharts `graph`(관리자 두뇌와 동일 계열) 또는 밸류체인형 HTML 플로우.  
- 노드에 `support_pct` 표시.  
- 호버 툴팁: 근거 2~5건 (`title`, `source`, `published`, `url`).

### 4.3 사이드/하단 패널 (노드 선택 시)

- 가정(assumptions)  
- 확인 지표(watch_metrics) — 현재값·방향만 (매크로/국면 API 재사용)  
- 관심 섹터 → `vc_key` / 시그널 필터 딥링크  
- 면책 한 줄

## 5. 데이터·점수

### 5.1 모델 (논리)

```
ScenarioTree
  id, as_of, disclaimer, source_rev
  nodes[]:
    id, parent_id, label
    support_pct          # sibling 정규화 후
    assumptions[]        # 짧은 문장
    watch_metrics[]      # e.g. regime, macro_bias, nasdaq, vix
    sector_keys[]        # valuechain key 또는 sectors 태그
    evidence[]           # { title, url, source, published, ticker? }
```

캐시: `kv` 키 예) `hypo:v1:{as_of}` 또는 해시. 일 1회·수동 시 재생성.

### 5.2 지지도 (룰 가중합 → sibling 정규화)

초기 가중(조정 가능):

| 성분 | 비중 | 입력 |
|---|---|---|
| 지표 일치 | 0.5 | regime / macro / 관련 지수 방향이 가정과 맞는지 |
| KB 근거 | 0.3 | 매칭 문서 수·신선도·(가능하면) digest 방향 |
| 사이클 정합 | 0.2 | `cycle.position` 주도섹터와 `sector_keys` 겹침 |

LLM은 **라벨·한 줄 요약·근거 문장 정리**만. `% 숫자는 룰이 산출**(요동·환각 방지).

### 5.3 템플릿 (P0 고정 3갈래 예시)

구현 시 문구·매핑은 큐레이션으로 확정. 예시:

1. **AI·CAPEX 지속** → 반도체·장비·전력 인프라 쪽 VC  
2. **정책·물가·소비 쪽 이동** → 소비재·내수  
3. **리스크오프** → 방어·배당·현금성

부모 하나 아래 세 자식, 지지도 합 100%.

## 6. API (초안)

| 메서드 | 경로 | 역할 |
|---|---|---|
| GET | `/api/hypothesis` | 현재 트리(캐시). `{ ready, as_of, tree, disclaimer }` |
| POST | `/api/hypothesis/refresh` | 수동 재점수(관리자 또는 로그인). 일일 수집 후에도 호출 |

- 읽기 전용 소비: `cycle`, `macro`, `regime`, `kb` 검색/엔트리.  
- **쓰지 않음**: `evaluate`, `signalcfg`, `bot.*`.

일일 훅: `_daily_kb_collect` 성공 후 `hypothesis.refresh()` 한 줄 (실패해도 본 루프 무시).

## 7. 기존 인프라 재사용

| 자산 | 용도 |
|---|---|
| `reference/cycle.py` | 국면·주도섹터 정합 |
| `reference/valuechain.py` | 섹터 키 → 국내 티커 |
| `kb` / `kb_entries.url` | 호버 근거 링크 |
| `kb_search.retrieve` | 가지별 근거 후보 (응답에 **url 노출 추가** 필요) |
| ECharts graph / VC 플로우 | 다이어그램 UI |
| 시그널 딥링크 | 관심 섹터 → 기존 리스트 |

제약(무리하지 않음):

- KB는 **전종목·실시간 크롤 아님** (lead/BUY/관심 + 거시, 하루 1회) — 합의와 일치.  
- KB에 `sector` 컬럼 없음 → 티커·거시·제목 접두·VC 매핑으로 우회.
