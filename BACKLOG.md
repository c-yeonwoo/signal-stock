# Signal Desk — MVP 기능 백로그

1단계 스캐폴딩 이후 붙일 핵심 기능 리스트. Signal APT의
["전체 기능 백로그"](https://github.com/c-yeonwoo/apt-signal) 문서 형식(우선순위·범위·리스크)을
따르되, 사용자가 정한 **phase1(비로그인 데이터/엔진 레이어)**·**phase2(로그인 사용자 기능)** 축을
그대로 유지한다. 우선순위: **P0** 없으면 서비스 정체성이 안 섬 · **P1** 핵심 차별화 · **P2** 확장.
범위: S(일) · M(수일) · L(1~2주) · XL(2주+).

## 0. 데이터 소스 결정 (P0, 선행 과제)

한국 주식부터 시작(요청사항: "정보를 얻기 쉬운 한쪽에서 시작"). 조사 결과 아래 조합이 **무료·공식**으로
기술적분석+기본적분석+백테스트+모의 자동매매를 전부 커버한다 — 별도 유료 API 없이 phase1 전체가
이 스택만으로 가능:

| 소스 | 용도 | 비고 |
|---|---|---|
| [KIS Developers(한국투자증권 오픈API)](https://apiportal.koreainvestment.com/intro) | 실시간 시세(WebSocket)·주문·**모의투자 계좌** | 무료, REST+실시간 소켓. **모의투자 리그**로 실제 돈 없이 자동매매봇 검증 가능 — 신뢰성・법적 리스크 둘 다 낮춤 |
| [KRX Data Marketplace / Open API](https://data.krx.co.kr/) | 과거 시세·거래대금·공매도·투자자별 매매동향(2010~) | 백테스트용 장기 시계열의 1차 소스 |
| DART 공시 API | 재무제표(PER/PBR/ROE 산출 원천) | brightdesk에서 이미 연동 이력 있음(`DART_API_KEY`) — 그대로 재사용 |
| [pykrx](https://github.com/sharebook-kr/pykrx) | 위 API 공백 시 스크래핑 폴백 | 비공식, 최후 수단으로만 |

미국주식은 위 스택이 자리잡은 뒤(phase1 후반~phase2) AlphaVantage 등으로 확장 — 지금은 범위에서 제외.

**진행 상황(2026-07-02, 키 없이 구현 착수 시점 확인)**: `pykrx`의 시장 전체/지수구성종목 계열
엔드포인트(`get_index_portfolio_deposit_file`, `get_market_cap_by_ticker`,
`get_market_ohlcv_by_ticker`, `get_market_ticker_list`)가 전부 KRX 응답 스키마 변경으로 깨져 있음을
직접 재현 확인(JSON 파싱 에러). 종목별 시계열(`get_market_ohlcv_by_date`)과 단일 종목명 조회는 정상.
→ **실제 코스피200 편입종목을 프로그래밍적으로 가져올 방법이 현재 없어서, 1차 구현은 잘 알려진
대형주 30종목을 임시 유니버스로 하드코딩**했다(`ingest/krx.py:_INTERIM_TICKERS`, 각 코드는
`get_market_ticker_name`으로 실시간 검증). **KRX Data Marketplace 키가 오면 §0 표의 1차 소스로
`universe()` 내부만 교체 — 상위 레이어는 영향 없음.** 진짜 코스피200 전체가 필요하면 이 교체가
최우선 후속 작업.

## Phase 1 — 비로그인 (데이터·엔진 레이어)

시그널 엔진 자체가 서비스 정체성이므로 이 구간이 최우선. `brightdesk`에서 이미 검증한 3팩터
가중 로직(기술 0.35·기본 0.30·KB 0.35, [NOTES-cherrypick.md](NOTES-cherrypick.md) 참고)을
그대로 재사용하되, KB(감성) 팩터는 데이터 소스가 갖춰지는 후순위로 미루고 기술+기본 2팩터로 시작.

| # | 기능 | 가치 | 우선 | 범위 | 의존/리스크 | 상태 |
|---|---|---|---|---|---|---|
| 1 | 기술적분석 엔진(RSI·MACD·MA) | 시그널 트리거의 절반 축 | P0 | M | KIS/KRX 시세 캐시 선행 | ✅ `signals/indicators.py` |
| 2 | 기본적분석 엔진(PER·PBR·ROE·매출성장·부채비율 스코어) | 나머지 절반 축 — "감이 아니라 성적표" | P0 | M | DART 공시 파싱 | ✅ `signals/fundamental.py` (DART 키 대기 — 그레이스풀 폴백 확인됨) |
| 3 | 통합 시그널(기술+기본 가중합) | 서비스 핵심 산출물 | P0 | M | #1·#2 | ✅ `signals/engine.py`, `/api/signals` |
| 4 | 백테스트 성적표(적중률·5일/20일 수익률) | Signal APT와 동일한 "신뢰 해자" — 유료 전제 조건 | P0 | L | #3 + 과거 시세(KRX) | ✅ 1차(기술 단독), `/api/backtest` — DART 확보 후 재무 반영 확장 예정 |
| 5 | 시장 국면(강세·과열·조정·약세) | 시그널에 국면 오버레이 — 개별 탭 아님(APT 패턴 유지) | P1 | M | 지수·거래대금 임계값 설계 |
| 6 | 하반기 전망(미래 지표 흐름 시나리오) | "다음엔 어떻게 될까"에 답 | P1 | L | ⚠️ 백테스트(과거 검증)와 성격이 다름 — 확정 예측 아닌 확률적 시나리오로만 표현. 표현 수위에 유사투자자문 규제 리스크(Signal APT 백로그 문서 §2-5 참고) |
| 7 | 실시간 자동투자봇(시그널 기반 모의투자 포트폴리오) | Track A(자동 운용) 데모의 핵심 — brightdesk에서 이미 시도했던 개념을 KIS 모의투자로 실화 | P0~P1 | XL | KIS 모의투자 계좌 연동 + 리스크 엔진(#8) 선행 필요 |
| 8 | 리스크 엔진(stop-loss/take-profit/trailing) | 자동매매봇 없이는 위험한 뼈대 — brightdesk 룰(−7%/+15%/−5%) 재검증 후 채택 | P0 | M | #7과 사실상 세트 |

**순서 권장**: 0(데이터소스) → 1·2(병렬 가능) → 3 → 4(성적표) → 8(리스크 룰) → 7(자동매매봇) → 5·6.
자동매매봇(#7)은 성적표(#4)로 신뢰를 먼저 증명한 시그널에만 태워야 함 — 순서를 앞당기지 않는다.

## 탭 배치 메모

현재 5탭(시그널·저평가·후보·관심비교·AI리포트) 기준:
- #5(국면)·#6(전망)은 **시그널 탭 안 오버레이**로(새 탭 아님 — APT 나침반 규칙 §6 "탭 무한증식 금지").
- #7(자동매매봇)·#9(시나리오)는 **관심·비교 탭 확장** 또는 신규 "포트폴리오" 탭 — 자동매매봇이 실제로
  붙는 시점에 결정.
- #10·#12(산업/히트맵)는 **저평가 탭 세그먼트**로 우선 검토.
- #11(뉴스)은 원래 Signal APT DESIGN.md 나침반엔 독립 탭이었으나, 우리는 시그널/후보 탭 내 패널로
  먼저 시도 — 콘텐츠 양이 많아지면 그때 탭 승격 검토.

## 규제/표현 유의 (Signal APT 백로그에서 이미 지적된 리스크, 주식도 동일 적용)

- #6(미래 전망)·#7(자동매매봇)·#9(시나리오 수익 예상)은 "확정 수익 보장"으로 읽히지 않게 확률·범위
  표현 강제, 면책 문구 상시 노출. 유사투자자문업 등록 요건 여부는 별도 확인 필요(법률 자문 대상,
  이 리포에서 결론 내지 않음).
